# Analyse du besoin client — Aéroworld

## 1. Introduction

**Aéroworld** est une entreprise française à dimension internationale, active depuis plus de cinquante ans dans l’industrie aéronautique. Elle intervient sur plusieurs activités clés : la conception, le développement, la fabrication et la maintenance d’avions, avec une présence dans de nombreux pays et des milliers d’employés répartis sur différents sites.

Dans le cadre de ce recrutement, Aéroworld recherche un profil de **Data Analyst chef de projet**, expérimenté, polyvalent et capable d’évoluer dans un environnement data complexe. Le profil attendu doit démontrer à la fois des compétences techniques solides, une compréhension des enjeux métier, une capacité de cadrage projet et une posture de consultant data.

L’objectif de cette analyse est d’identifier les besoins exprimés par Aéroworld, de les structurer en exigences exploitables, puis de montrer comment mon portfolio permet d’y répondre à travers des projets concrets, des compétences démontrées et des preuves associées.

---

## 2. Contexte client

Aéroworld évolue dans un secteur industriel exigeant, où la donnée joue un rôle stratégique. L’entreprise génère et exploite des volumes importants de données issues de sources multiples :

- essais en vol ;
- opérations en temps réel ;
- capteurs embarqués ;
- systèmes de maintenance ;
- données clients ;
- données opérationnelles ;
- données de conception.

Ces données doivent être collectées, stockées, consolidées, analysées et sécurisées afin de produire de la valeur pour l’entreprise.

Les enjeux data identifiés sont les suivants :

- gérer des données massives à grande échelle ;
- mettre en place des systèmes robustes de stockage de type Data Lake ;
- intégrer et harmoniser des données issues de plusieurs départements et systèmes ;
- garantir la fiabilité, la sécurité et la confidentialité des données ;
- exploiter les données pour améliorer la conception des avions ;
- optimiser les performances opérationnelles ;
- prévoir les besoins en maintenance ;
- soutenir l’innovation via l’apprentissage automatique et l’intelligence artificielle ;
- transformer les données en informations stratégiques et exploitables.

Le besoin client dépasse donc la simple production d’analyses. Aéroworld recherche un profil capable de comprendre les enjeux, structurer le besoin, produire des livrables fiables, accompagner les utilisateurs et adopter une posture de conseil.

---

## 3. Objectif de l’analyse du besoin

Cette analyse vise à traduire le cahier des charges client en exigences opérationnelles pour le portfolio.

Elle poursuit quatre objectifs principaux :

1. **Identifier les attentes explicites du client**  
   Recenser les compétences techniques, métier et soft skills attendues dans le brief Aéroworld.

2. **Structurer les besoins en use cases data**  
   Organiser les attentes autour de cas d’usage tels que la gestion de données massives, l’interopérabilité, la maintenance prédictive, la sécurité des données, l’aide à la décision ou l’accompagnement des équipes.

3. **Évaluer l’adéquation du portfolio**  
   Mettre en relation les attentes client avec les projets présentés dans le portfolio.

4. **Produire des indicateurs de couverture**  
   Construire un dashboard permettant de mesurer le taux de couverture des besoins client et le niveau global d’alignement du profil.

## 5. Structuration des attentes client dans le dashboard

Les attentes client ont été structurées dans une table dédiée : **ClientRequirements**.

Cette table contient **64 compétences attendues**, réparties en trois catégories :

- **Technique** ;
- **Métier** ;
- **Soft skill**.

Chaque compétence est associée à :

- un identifiant unique ;
- une catégorie ;
- un domaine de compétence ;
- un niveau d’importance ;
- un use case Aéroworld.

Les use cases Aéroworld sont organisés dans une table dédiée : **AeroworldUseCases**. Ils représentent les grands scénarios data identifiés dans le cahier des charges client : gestion de données à grande échelle, Data Lake, interopérabilité, sécurité, maintenance prédictive, aide à la décision, documentation, accompagnement et posture consultant.

Une table de preuves, **RequirementEvidence**, relie ensuite les exigences client aux projets du portfolio. Cette table permet de mesurer le niveau de couverture de chaque attente par les projets présentés.

## 6. Méthode d’évaluation de l’adéquation

L’adéquation entre mon portfolio et le besoin Aéroworld est évaluée à partir de deux indicateurs principaux.

### Taux de couverture

Le taux de couverture mesure la part des attentes client couvertes par au moins un projet du portfolio.

Une attente est considérée comme couverte lorsqu’elle dispose d’un niveau de couverture suffisant dans la table des preuves.

### Global alignment

Le **Global alignment** est l’indicateur final d’évaluation de l’adéquation du profil au besoin client.

Il va plus loin qu’un simple taux de couverture, car il combine deux dimensions :

- l’importance de chaque attente pour Aéroworld ;
- la force de la preuve apportée par les projets du portfolio.
